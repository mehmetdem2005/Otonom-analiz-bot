BİLEŞENLER (6 Modül)
────────────────────────────────────────────────────────────

DB-1: SQLite → PostgreSQL Geçişi
  Mevcut durum: JSONL dosyaları (memory/, log/, trace/)
  Hedef: asyncpg + PostgreSQL 15+
  Araçlar:
    - asyncpg: async PostgreSQL sürücüsü
    - Alembic: şema migration yönetimi
    - SQLModel veya SQLAlchemy 2.0: ORM katmanı
  Tablolar:
    - memory_records (id, ts, kind, path, outcome, hint, embedding vector)
    - trace_records (id, session_id, ts, event_type, payload jsonb)
    - quality_snapshots (id, ts, metric_name, value, metadata jsonb)
    - plan_items (id, text, status, section, created_at, updated_at)
  Geçiş stratejisi:
    1. JSONL okuyucuyu interface olarak soyutla (adapter pattern)
    2. PostgreSQL adapter yaz, aynı interface uygula
    3. Dual-write geçiş dönemi (JSONL + PG paralel)
    4. JSONL'yi arşivle, PG'ye geç
  Durum: YAPILMADI

DB-2: Vektör İndeksi (Semantik Hafıza)
  Mevcut durum: Python dict'te in-memory embedding (yavaş, geçici)
  Hedef: pgvector (PostgreSQL eklentisi) veya FAISS kalıcı index
  Seçenek A — pgvector (tavsiye edilir):
    - PostgreSQL'e vector tip eklentisi
    - CREATE INDEX USING ivfflat (embedding vector_cosine_ops)
    - Sorgu: SELECT * FROM memory ORDER BY embedding <=> $1 LIMIT 10
    - Avantaj: tek veritabanı, ACID garantisi, yedekleme basit
  Seçenek B — FAISS:
    - Disk'e kaydedilen FAISS IndexFlatIP
    - Yüksek hacimde daha hızlı ANN araması
    - Dezavantaj: PG ile senkronizasyon gerekli
  Öneri: pgvector ile başla, ölçek gerektirirse FAISS'e geç
  Durum: YAPILMADI

API-1: FastAPI Üretim Katmanı
  Mevcut durum: basit FastAPI, hata işleme minimal
  Hedef: üretim kalitesi API
  Eklenecekler:
    - Request ID middleware (UUID, X-Request-ID header)
    - Yapılandırılmış hata yanıtları (ErrorResponse model)
    - Rate limiting (slowapi veya custom token bucket)
    - JWT tabanlı kimlik doğrulama (dashboard + API)
    - Prometheus metrikleri (prometheus-fastapi-instrumentator)
    - /health ve /ready endpoint'leri (Kubernetes probe)
    - OpenAPI dokümantasyonu otomatik (zaten var, geliştir)
    - CORS yapılandırması (şu an açık)
  Durum: YAPILMADI

UI-1: Gerçek Zamanlı WebSocket Dashboard
  Mevcut durum: polling (setInterval 5s), vanilla JS, canvas charts
  Hedef: WebSocket + Chart.js
  Mimari:
    - FastAPI WebSocket endpoint: /ws/dashboard
    - Server push: agent events, memory updates, risk scores
    - Chart.js kütüphanesi (CDN) → gerçek interaktif grafikler
    - Responsive tasarım (Tailwind CSS CDN)
    - Dark/light mode toggle
    - Gerçek zamanlı log stream (WebSocket üzerinden)
  WebSocket protokolü:
    {
      "type": "memory_update" | "agent_event" | "risk_update" | "log",
      "ts": float,
      "data": {}
    }
  Durum: YAPILMADI

TOOL-1: Gerçek Araç Entegrasyonları
  Mevcut durum: bazı araçlar simülasyon veya sınırlı

  Web Araması:
    - SerpAPI veya Tavily API entegrasyonu
    - Fallback: DuckDuckGo (ücretsiz, sınırlı)
    - Sonuç önbellekleme (Redis veya bellek içi TTL cache)

  Kod Çalıştırma Sandbox:
    - Docker container'da izole çalıştırma
    - CPU/memory/zaman limiti (--memory=256m --cpus=0.5)
    - Ağ erişimi kapalı (--network=none)
    - Geçici dosya sistemi (--tmpfs /tmp)
    - Python executor: docker run --rm python:3.11-slim

  Twitter/X OAuth 1.0a:
    - tweepy kütüphanesi ile gerçek OAuth akışı
    - Token yönetimi (güvenli .env, rotation)
    - Rate limit farkındalığı (API v2 limitleri)

  Dosya Sistemi:
    - seccomp profili ile sistem çağrısı kısıtlama
    - chroot jail veya bubblewrap sandbox

  Durum: YAPILMADI

OPS-1: Docker Compose + Gözlemlenebilirlik
  Mevcut durum: tek process, manuel başlatma
  Hedef: tam orkestrasyon

  docker-compose.yml servisleri:
    - agent: FastAPI ajan servisi (web_arayuzu.py)
    - db: PostgreSQL 15 (volume ile kalıcı)
    - redis: önbellekleme + rate limit state
    - prometheus: metrik toplama
    - grafana: dashboard görselleştirme
    - loki: log agregasyonu
    - promtail: log toplama ajanı

  Yedekleme:
    - pg_dump cron (günlük, S3 veya yerel)
    - JSONL arşiv sıkıştırma (haftalık)

  Health Checks:
    - /health: servis canlı mı?
    - /ready: DB bağlantısı var mı?
    - Kubernetes liveness/readiness probe uyumlu

  Alerting:
    - Prometheus AlertManager kuralları
    - Kritik eşikler: rollback_rate > 0.3, API latency > 5s

  Durum: YAPILMADI
